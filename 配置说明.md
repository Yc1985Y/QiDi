## vivo API 配置

Flutter 工程不在源码中硬编码 API 凭据。真实解析必须通过 `--dart-define` 传入：

```powershell
flutter run --dart-define=VLM_APP_ID=你的AppID --dart-define=VLM_API_KEY=你的AppKey
```

构建 APK 时同理：

```powershell
flutter build apk --debug --dart-define=VLM_APP_ID=你的AppID --dart-define=VLM_API_KEY=你的AppKey
```

可选参数：

- `VLM_MODEL_NAME`
- `VLM_API_ENDPOINT`
- `VLM_OCR_ENDPOINT`

当前默认接口：

- Chat：`https://api-ai.vivo.com.cn/v1/chat/completions`
- OCR：`https://api-ai.vivo.com.cn/ocr/general_recognition`
- 默认模型：`Volc-DeepSeek-V3.2`

如果未传入 `VLM_API_KEY`，解析服务会明确报错，不会使用 mock、demo 或本地规则兜底。

## Android 构建说明

当前 Flutter 工程使用：

- `receive_sharing_intent: 1.8.1`
- `add_2_calendar: 3.1.1`
- `flutter_local_notifications: 22.0.1`
- `pdf: 3.13.0`
- `open_filex: 4.7.0`

说明：

- `receive_sharing_intent 1.9.0` 要求 compileSdk 37，但当前 sdkmanager 在本机安装的是 `platforms\android-37.0`，Gradle 查找 `android-37` 会失败。因此本工程固定到仍支持同一 Dart API、可在当前 SDK 下构建的 `1.8.1`。
- `flutter_local_notifications` 要求 app 启用 core library desugaring，已在 `android/app/build.gradle.kts` 中配置 `desugar_jdk_libs:2.1.4`。
- 根 `android/build.gradle.kts` 已按插件类型统一 JVM target：主 app 和现代插件用 17，老 KGP 插件用 11。
