## 本轮已落地

- 新增系统分享接收链路：
  - Android Manifest 已注册 `SEND text/*`、`SEND image/*`、`SEND_MULTIPLE image/*`。
  - Flutter 侧新增 `ShareReceiveService`，冷启动和热启动分享均会进入现有 `parseInput`。
  - 文本分享走 `SourceType.shareText`，图片分享走 `SourceType.shareImage`，图片附带文字时一并送入 OCR + 解析链路。
- 新增系统日历链路：
  - `IntegrationService.addEventToCalendar(...)` 使用 `add_2_calendar` 调系统日历确认页。
  - `confirmNotice(...)` 现在会保存时间线、排本地提醒，再调系统日历；日历失败会明确显示错误，不静默伪装成功。
- 新增时间线 PDF 导出：
  - `TimelineExportService` 使用 `pdf` 生成真实 PDF。
  - PDF 写入应用文档目录 `exports/timeline-*.pdf`。
  - 已加入 `NotoSansSC-Regular.otf` 字体资产，保证中文标题、地点、说明可渲染。
  - 时间线页顶部新增 PDF 导出按钮。
- 继续保留：
  - vivo OCR + chat completions 真实解析。
  - 无 API key 时真实报配置错误，不做 mock/demo/本地兜底。
  - 本地提醒、语音、TTS、地图、复制、系统分享出站能力。

## 本轮构建修复

- `receive_sharing_intent` 固定为 `1.8.1`，避免本机 `android-37.0` 与 Gradle `android-37` 查找不一致导致构建失败。
- `android/build.gradle.kts` 增加 JVM target 分流：
  - 主 app 和现代插件：JVM 17。
  - `flutter_tts`、`receive_sharing_intent`、`speech_to_text`：JVM 11。
- `android/app/build.gradle.kts` 启用 core library desugaring，并加入 `desugar_jdk_libs:2.1.4`。
